# Project Behavior Rules

- **Git Submissions**: After making adjustments or code changes, ALWAYS stage and commit the changes to the local Git repository.
- **GitHub Pushing**: DO NOT push commits to GitHub (`git push`) unless the user explicitly instructs you to do so in the chat.
- **Git 提交规则**：每次完成代码调整或功能实现后，必须将更改提交到本地 Git 仓库（`git commit`）。
- **推送 GitHub 规则**：除非用户在对话中明确下达推送指令，否则绝对不能自动或擅自推送到 GitHub 远程仓库（不执行 `git push`）。
